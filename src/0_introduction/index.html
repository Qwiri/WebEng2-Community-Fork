- Frage an die Studenten geben. - Vorschläge auf der Tafel/Flipchart sammeln.
- Versioning im Sinne von verschiedene Versionen eines Clients rufen die selbe Serverschnittstelle auf - Webanwendung wird immer neu ausgeliefert. Keine veralteten Versionen.
- unterschiedliche Plattformen (Mobile und Web mit verschiedenen Betriebssystemen) - unterschiedliche Systemumgebungen, unterschiedliche Entwicklungsumgebungen - Web Browser funktionieren theoretisch alle gleich. In der Praxis unterscheiden sie sich jedoch (in Details).
- Früher gab es Anwendungen auf CD's heute läd man diese im Internet herunter. - Anwendungen können übers Web ohne Probleme ausgeliefert werden, die schön und Benutzbar sind. - Client Anwendungen existieren noch auf Smartphones. Navigation der Betriebssystemen ist unterschiedlich.
- Anteile variieren. - Kommt auf das Thema an.
- Umsetzung mit erlernten Technologien - wir schauen mal wie weit wir kommen, mit dem was wir schon können - früher nutze JSF JSP, seit JSF 2 gibt es eine eigenes Rendering (facelets -> erweiterte servlets). - JSF - Validierung - Konvertierung - Events -> Userevents aus dem Frontend
- Aufteilung in Frontend und Backend - Rich Clients sind häufig in Frontend und Backend geteilt, da das Frontend bereits eine eigene Anwendung ist. - Das Backend liefert normalerweise nur noch Daten und führt Geschäftslogik aus.
- Warum Angular? - weit verbreitet - werdet ihr bestimmt in Projekten mal sehen - ich kenne mich damit aus - Warum Spring? - wichtig für das Arbeitsleben - wird euch wahrscheinlich mal unterkommen - wenig Zeit für das Setup - weniger auf die Technik konzentrieren
- Nutzer müssen identifiziert werden - brauchen verschiedene Rechte - dürfen nur auf bestimmte Daten zugreifen - müssen evtl. für Dienstleistungen bezahlen - Benutzerverwaltung ist kompliziert - Bitte nicht selbst machen - OAuth2 - weil kompliziert bitte Standards verwenden - und best practices einhalten
- Wie Gebäude müssen auch Webanwendungen einbruchssicher sein - Vertraulichkeit, Integrität und Verfügbarkeit - Fokus liegt auf den Features, nicht auf den Qualitätskriterien - Messbarkeit ist schwierig
