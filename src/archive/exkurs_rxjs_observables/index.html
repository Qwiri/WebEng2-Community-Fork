RxJS Observables

Events

  • Nutzerinteraktionen sind Events
  • Beispiel: "onclick", "onchange", etc.
  • um auf Events zu reagieren wird ein Listener benötigt
                document.addEventListener(
    'click',
    () => console.log('Clicked')
);

Observables

  • Observables vereinfachen den Umgang mit Events
  • RxJS verfolgt den Functional Reactive Ansatz
  • asynchrone Event Streams werden wie Arrays behandelt
    • Pipes and Filters
  • der Functional Reactive Ansatz soll die Architektur eines interaktiven Systems verbessern
  • klassischerweise reagiert ein interaktives System immer auf Events des Nutzers
  • diese Events werden dann bearbeitet und verändern anschließend den State des Systems
  • der neue State des Systems macht sich anschließend wieder über z.B. eine UI sichtbar
  • Beispielsweise kann man sich ein Radio vorstellen
    • der Nutzer dreht an der Volume
    • anschließend wird ein Event gesendet: "Musik leiser"
    • dieses Event beeinflusst den internen State des Radios
    • der State wird "hörbar" durch eine Änderung der Lautstärke

Observables

  • es wird auf einem asynchronen Event Stream subscribed
  • auf den Events können Operationen ausgeführt werden
  • das dies asynchron passiert ist erstmal irrelevant
  • quasi wie Streams in Java
                fromEvent(document, 'click')
  .pipe(scan(count => count + 1, 0))
  .subscribe(count => console.log(`Click count ${count}`));
  • RxJS bietet mit Observables einen guten Handwerkskasten für FRP
  • wir nutzen für unser State Management in unserer Anwendung auch RxJS
