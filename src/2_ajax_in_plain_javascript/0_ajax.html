Multi Page Application (Erinnerung)

        
          @startuml
          activate Client
          Client -> Server: Request (GET)
          deactivate Client
          activate Server
          Server -> Client: Response (HTML Page)
          deactivate Server
          activate Client
          Client -> Server: Request (Post)
          deactivate Client
          activate Server
          Server -> Client: Response (HTML Page)
          deactivate Server
          activate Client
          @enduml
  • Jedes mal, wenn der Nutzer andere Daten sehen möchte, auf eine andere Seite navigieren oder ein Formular abschicken will, wird ein Request ans Backend geschickt und er erhält als Antwort eine neue HTML Seite.
  • Während der Server die Antwort vorbereitet, kann der Nutzer nicht mit der Seite interagieren.
  • Erst wenn der Server diese neue Seite zurückschickt, kann der Browser das Ergebnis anzeigen.

Was ist Ajax?

  • Es ist keine Programmiersprache!
  • Es handelt sich um ein Programmierprinzip
  • Daten anzeigen, ohne eine neue HTML Seite auszuliefern
  • Die Idee ist dem Nutzer andere Daten anzuzeigen, ohne eine komplett neue HTML Seite zu laden.

  • Kommen die Daten zurück, muss das HTML manipuliert werden, damit die Daten angezeigt werden.
